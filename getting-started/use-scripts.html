Create task from a script

Take existing scripts and use them in PE as Tasks

Description

You’ll create and convert a bash/powershell script into a Puppet Task, which can then be used to perform ad-hoc action across your infrastructure in Puppet Enterprise.

In this lab you will:

  • Create a simple script
  • Create task metadata
  • Add to control repo

Important: The target node must have the pre-reqs neccessary to run the script for a given language.

Video walkthrough

Labs

1

Create a simple script

  1. Create script with the name corresponding to platform:

    2. Linux: lin_info.sh

    Windows: win_info.ps1

    • Linux
    • Windows 
                          
                        #!/bin/bash
                        info=$(lscpu)
                        echo -e "System Info:\n \n$info"
                        echo "Your custom parameter was: ${PT_myparam1}"                         
                      
                    
                          
                        Param($myparam1)
                        $systeminfo = systeminfo
                        Write-Output "System Info:" $systeminfo
                        Write-Output "Your custom parameter was: ${myparam1}"

2

Add script to task directory

  1. Login to Gitlab web interface and navigate to your Control Repo
  2. Navigate to the site-modules directory
  3. Click the + icon in the navigation bar at the top and select New directory
  4. Make sure Start a new merge request with these changes is not checked
  5. Click Create directory
  6. In the lab directory, repeat the same process to create a directory named tasks
  7. Click the + icon in the navigation bar at the top and select Upload file then choose your script

3

Find your new Task in Puppet Enterprise

  1. Switch back to the Puppet Enterprise web interface
  2. Navigate to ORCHESTRATION, Tasks and click Run a task on the top right hand corner
  3. Click on the dropdown box under Task and locate your lab::lin_info or lab::win_info tasks
  4. Click View task metadata. Notice the tasks have virtually no documentation or explanation at this point

4

Create Task metadata

As seen above, it’s easy to add scripts as Tasks to Puppet Enterprise. Simply put your scripts in a tasks subdirectory of any module, the simplest approach for a new module being a new directory under site-modules in your Control Repo. But Tasks become a whole lot easier to use if they provide builtin explanations of what they do and how they can be used. This, and more, is what we can achieve with Task metadata.

Task metadata is a JSON file that accompanies a script, by having the same name as the script, but with a JSON file extension. The file contains descriptions for the Task itself, as well as for any of the Task’s parameters. It also allows you to easily control which parameters are mandatory vs optional, and what input values/types are allowed for each parameter. Task metadata also enables a number of more advanced capabilities, which you can find documented here.

  1. Create task metadata file with the name corresponding to each script:

    2. Linux: lin_info.json

    Windows: win_info.json

    • Linux
    • Windows 
                              
                            {
                              "description": "Displays system information for a given Linux node via lscpu command",
                              "input_method": "environment",
                              "parameters": {
                                      "myparam1": {
                                            "description": "Your custom parameter",
                                            "type": "Any"
                                      }
                                  }
                              }                       
                          
                        
                              
                            {
                              "description": "Displays system information for a given Windows node via systeminfo command",
                              "input_method": "powershell",
                              "parameters": {
                                      "myparam1": {
                                            "description": "Your custom parameter",
                                            "type": "Any"
                                      }
                                  }
                              }
  2. Now follow the same procedure as outlined in Step 2, to add the JSON files to the tasks subdirectory of the lab module in your Control Repo.
  3. If, after adding the JSON files, you checked on your Tasks in Puppet Enterprise again, you’ll notice that your Tasks now have proper descriptions and parameter guidance.
