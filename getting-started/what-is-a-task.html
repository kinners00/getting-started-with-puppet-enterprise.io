What is a task?

Overview

Puppet Tasks enable you to easily automate ad hoc tasks and eliminate manual work across your infrastructure and applications.

Tasks comprise of two key components:

  • Scripts

    • You can choose any script, written in any language to become a task in Puppet Enterprise, providing that your target node that run the script. This means you can leverage your existing investments and workflows when using Puppet tasks

  • Task Metadata

    • Tasks become a whole lot easier to use if they provide builtin explanations of what they do and how they can be used. This, and more, is what we can achieve with Task metadata.

    Task metadata is a JSON file that accompanies a script, by having the same name as the script, but with a JSON file extension. The file contains descriptions for the Task itself, as well as for any of the Task’s parameters. It also allows you to easily control which parameters are mandatory vs optional, and what input values/types are allowed for each parameter. Task metadata also enables a number of more advanced capabilities, which you can find documented here.

Where are they stored?

Tasks will be recognized and available in Puppet Enterprise if they are contained within a module. To do this in it's simplest form, tasks can be added under module directory of your choice in the site-modules directory of the control repository.

Tasks are automatically namespaced with the format MODULENAME::TASKNAME.

The example task is named maintenance::log_rotate

You can find the directory structure for our task below: 

                      

                      ├─ controlrepo/
                          └─ site-modules/    
                              └─ maintenance/      
                                  └─ tasks/
                                      └─ log_rotate.sh    <-- Script      
                                      └─ log_rotate.json  <-- Task
