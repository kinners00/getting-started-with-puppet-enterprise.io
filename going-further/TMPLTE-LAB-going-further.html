Use Existing Scripts

Run scripts in any language across infrastructure nodes

Description

With PE you can take existing scripts and bring them into PE to enable to you to quickly and easily begin automating at scale. Scripts can use any coding language for example bash, powershell, python etc.

Important

The target node must have the pre-reqs neccessary to run the script for a given language.

Labs

1

Create script

- Create a powershell or bash script with the following name:

Windows: timesync.ps1

Linux: timesync.sh

Make sure that your script has the correct permissions and is executable.

If you already have a script for Windows or Linux that you know works on your test nodes, feel free to use that instead.

2

Add script to control-repo

- Navigate to your control repo in Visual Studio Code

- Create a new folder in /site-modules named lab

- Create a folder in /lab named tasks

- Copy your script to /tasks

3

Deploy Puppet Code

Note

You can skip this step if you already have webhooks configured that automatically trigger a puppet code deploy

Run the command below

                  
                    puppet code deploy --all --wait

If successful, your puppet server should have a new version of code from your control repo. This means that your script should be available to use in puppet enterprise!

4

Run Script

- From the puppet console, navigate to Tasks and then click Run Task

- Under the Task click the dropdown Enter a task to run and type lab::timesync and choose your task

- Scroll down the page and click the dropdown Select a target type

- Choose Node group and then select or type the Node group name that contains the desired target for this task. Click Select and then Run task.

You should see your task run successfully on your node(s) !
